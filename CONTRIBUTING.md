# Contributing to Migration and Backup Tool

Thank you for your interest in contributing to the Migration and Backup Tool! We welcome all contributions, including bug reports, feature requests, and code improvements.

## Code of Conduct

Please be respectful and professional in all interactions.

## How to Contribute

### Reporting Bugs

- Use the GitHub issue tracker.
- Provide a clear and descriptive title.
- Describe the steps to reproduce the issue.
- Include information about your environment (OS, Go version, database version).

### Feature Requests

- Use the GitHub issue tracker.
- Describe the use case and why the feature is needed.
- Provide examples of how it should work.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your changes.
3. Make your changes and ensure tests pass.
4. Submit a pull request with a clear description of your changes.

## Development Setup

1. Install Go 1.26 or later.
2. Clone the repository.
3. Run `make build` to build the tool.
4. Run `make test` to run tests.

## Style Guide

- Follow standard Go conventions (`gofmt`).
- Use descriptive names for variables and functions.
- Write unit tests for new features and bug fixes.
